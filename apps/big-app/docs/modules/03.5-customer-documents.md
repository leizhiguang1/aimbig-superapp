# Module 03.5 — Customer Documents (E-Documents)

## Overview

E-Documents is a unified panel that lives in three places:
- Customer detail → Documents tab
- Appointment detail → Documents tab
- (Future) Standalone document browser

It shows **all document types** for a customer in one list. Currently implemented types:

| Type | Status | Notes |
|------|--------|-------|
| Files (image/PDF) | ✅ Live | Upload, preview, download, delete |
| Letters | ✅ Live | Template picker + rich text editor, print page |
| Forms | ⏳ Deferred | Consent form structure documented below |
| Collages | ⏳ Deferred | — |
| Scan | ⏳ Deferred | — |

---

## Config location

**Config → Clinical Features → E-Documents tab**

In KumoDent: single tab with two side-by-side panels — "Form Listing" (left) and "Letters" (right).

In big-app: separate routes:
- Letters config → `/config/letter-templates` (under Clinical Features in rail)

## Letters

### DB table: `letter_templates`

| Column | Type | Notes |
|--------|------|-------|
| id | uuid PK | |
| brand_id | uuid FK → brands | Tier-A, required |
| name | text | Display name, e.g. "Referral Letter" |
| body_html | text | Rich HTML; merge fields in `{{snake_case}}` |
| is_active | bool | Only active templates appear in picker |
| created_at / updated_at | timestamptz | Standard |

### Merge fields (our format)

| Field | Renders as |
|-------|-----------|
| `{{customer_name}}` | Full name (first + last) |
| `{{customer_id_number}}` | NRIC / passport number |
| `{{customer_age}}` | e.g. "32 years 4 months" |
| `{{date}}` | Today's date (dd MMMM yyyy) |

KumoDent also supports: `CUSTOMER_FIRSTNAME`, `CUSTOMER_LASTNAME`, `CUSTOMER_SALUTATION`, `CUSTOMER_GENDER`, `CUSTOMER_CONTACT_NUMBER`, `CUSTOMER_ADDRESS`, `CUSTOMER_DOB`, `SUBDOMAIN_NAME`, `BUSINESS_NAME`, `OUTLET_NAME`, `OUTLET_ADDRESS`, `APPT_REF`, `APPT_STARTTIME`, `EMPLOYEE_NAME`, `EMPLOYEE_ROLE`. Add these to our merge pipeline when needed.

### Default seed template: Referral Letter

Seeded for every brand via `WHERE NOT EXISTS` guard. Content (using our merge syntax):

```html
<p>Dear *Insert Doctor Name*,</p>
<p><br></p>
<p>Re: {{customer_name}}, {{customer_age}} ({{customer_id_number}})</p>
<p><br></p>
<p>Thank you for seeing {{customer_name}}, {{customer_age}}, regarding *insert patient complications*</p>
<p>Current medication provided include *insert medication and doses provided*</p>
<p><br></p>
<p>Could I request that a treatment plan timeline be forwarded to me to assist with patient preparation as well as any feedback regarding this patient's test results or current medicines.</p>
<p><br></p>
<p>Thank You</p>
```

### Print page

`/documents/letters/[id]` — public route (auth-gated), renders letterhead from brand/outlet + `dangerouslySetInnerHTML` of saved `letter_body_html`. Print button triggers `window.print()`.

## Forms (Deferred — structure documented for future build)

### KumoDent reference: Consent Form

KumoDent's form builder uses **fixed section types** rather than arbitrary fields. The Consent Form has exactly two sections:

#### Section 1: Open Text (`fixed_section: "open_text"`)

Free-text area with a header. KumoDent default content:

```
I agree to receive consultation, assessment, and/or services provided by
this establishment. The nature and purpose of the services have been
explained to me, including any possible risks or side effects. I understand
I can ask questions at any time and may withdraw my consent if I choose to
discontinue.

I confirm that the personal and health information I've provided is
accurate and complete.
```

#### Section 2: Terms & Conditions (`fixed_section: "tnc"`)

Bullet list. KumoDent default content:

```
- Consent is given voluntarily.
- Results may vary for each individual.
- I will follow any advice or aftercare provided.
- I will inform the provider of any changes in my health or condition.
- Photos may be taken for documentation purposes with privacy protected.
```

### Planned DB tables (not yet created)

```sql
-- form_templates: brand-level form definitions
create table form_templates (
  id uuid primary key default gen_random_uuid(),
  brand_id uuid not null references brands(id) on delete restrict,
  name text not null,
  is_active bool not null default true,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

-- form_template_sections: ordered sections within a form
create table form_template_sections (
  id uuid primary key default gen_random_uuid(),
  form_template_id uuid not null references form_templates(id) on delete cascade,
  sort_order int not null default 0,
  section_type text not null check (section_type in ('open_text', 'tnc', 'signature', 'checkbox', 'short_answer')),
  title text,       -- shown above the section
  body_html text,   -- default/prefill content (rich text for open_text/tnc)
  required bool not null default false,
  created_at timestamptz not null default now()
);

-- customer_form_responses: filled-in form linked to customer + optional appointment
create table customer_form_responses (
  id uuid primary key default gen_random_uuid(),
  customer_id uuid not null references customers(id) on delete restrict,
  appointment_id uuid references appointments(id) on delete set null,
  form_template_id uuid references form_templates(id) on delete set null,
  form_name text not null,  -- snapshot at time of creation
  signed_at timestamptz,
  signed_by_name text,
  sections jsonb not null default '[]',  -- [{type, title, response_html, signed}]
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);
```

### Default seed: Consent Form

When forms are built, seed one Consent Form per brand with the two fixed sections above. Guard with `WHERE NOT EXISTS` on `(brand_id, name)`.

## Files

Stored in Supabase Storage bucket `documents`. Path: `{brand_id}/{customer_id}/{uuid}-{filename}`.

Upload flow:
1. `requestCustomerDocumentUploadUrlAction` → signed upload URL
2. Client uploads directly to Supabase Storage
3. `createCustomerDocumentAction` → inserts `customer_documents` row

Supported MIME types: `image/jpeg`, `image/png`, `image/webp`, `application/pdf`. Max 20 MB.
