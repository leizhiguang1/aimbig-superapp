# Module 15 — Manual Transactions

## Purpose

A Manual Transaction (MT) is a free-form, customer-linked record that lets
staff charge or credit items (services or products) outside the normal
appointment → Sales Order flow.

**Common use-cases observed from reference app:**
- Ad-hoc service charges not tied to an appointment
- Recording an outstanding balance settlement (e.g. "payment for SO-000123")
- Giving a customer a complimentary service recorded for audit

MTs are **NOT** appointment-linked and **NOT** a Sales Order. They are a
sibling record to `sales_orders`, not a child.

## Data model

| Column | Type | Notes |
|---|---|---|

| `outlet_id` | uuid FK → outlets | |
| `customer_id` | uuid FK → customers | |

| `cancelled_by` | uuid FK → employees | Nullable |
| `cancelled_at` | timestamptz | Nullable |
| `cancel_reason` | text | Nullable |
| `created_by` | uuid FK → employees | Nullable (set to current employee) |
| `created_at` | timestamptz | |
| `updated_at` | timestamptz | |

### `manual_transaction_items` (line items)

| Column | Type | Notes |
|---|---|---|
| `id` | uuid PK | |
| `manual_transaction_id` | uuid FK → manual_transactions | Cascade delete |
| `item_type` | text | `service` or `product` |
| `service_id` | uuid FK → services | Nullable |
| `inventory_item_id` | uuid FK → inventory_items | Nullable |
| `item_name` | text | Denormalized for audit trail |

| `unit_price` | numeric(12,2) | Staff can override at create time |

### `mt_counters`

Per-outlet sequence table (same pattern as `so_counters`). Generates the
`MT-000001` code via a `BEFORE INSERT` trigger on `manual_transactions`.

## RPCs

| Function | Purpose |
|---|---|
| `create_manual_transaction(p_outlet_id, p_customer_id, p_created_by, p_remarks, p_items jsonb)` | Atomically inserts header + all line items. Returns `{id, code}`. |
| `cancel_manual_transaction(p_mt_id, p_cancelled_by, p_reason)` | Marks status=cancelled, sets cancelled_by/at/reason. Raises if already cancelled. |

## Create flow (3 steps)

```
[Topbar Receipt button]
        ↓
  ManualTransactionDialog
        ↓ (+ New)
  Step 1: Select Customer
    - Search by name / code / phone
    - If only one outlet: outlet pre-selected from topbar context
    - If multiple outlets: outlet selector shown
        ↓ (click customer row)
  Step 2: Select Items
    - Services tab: lists all active brand services (name, SKU, price)
    - Products tab: placeholder (deferred — products are Phase 2)
    - Click service to add to cart
    - Cart shows qty + price, both editable
        ↓ (Continue)
  Step 3: Review + Remarks
    - Summary of customer + line items + total
    - Optional remarks field (max 500 chars)
        ↓ (Create Transaction)
  → create_manual_transaction RPC
  → navigates to Detail view of new MT
```

---

## Cancellation flow

From the Detail view, active MTs show a **Cancel** button. This opens a
nested `Dialog` requiring a reason (free text). On confirm:
- Calls `cancel_manual_transaction` RPC
- Updates `status = 'cancelled'`, sets `cancelled_by`, `cancelled_at`,
  `cancel_reason`
- Returns to the list view

---

## Invoice code format

`MT-NNNNNN` where N is a 6-digit zero-padded sequence **per outlet**.
Same pattern as `SO-000001` (sales orders). Code is assigned by a
`BEFORE INSERT` trigger; app code never writes it directly.
